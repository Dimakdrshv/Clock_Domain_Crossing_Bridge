# CDC Bridge

Лёгкий и переиспользуемый CDC-мост (Clock Domain Crossing) для FPGA-проектов.

Проект демонстрирует безопасную передачу данных между асинхронными тактовыми доменами с использованием стандартных методов синхронизации.

## Проблема CDC

Проблема Clock Domain Crossing (CDC) возникает при передаче данных между различными тактовыми доменами.

Основная сложность заключается в том, что данные, сформированные в одном тактовом домене, могут привести к нарушению временных требований триггеров на стороне приёма. В результате этого возможно возникновение метастабильного состояния.

Именно поэтому для **небольших** потоков данных возможно использовать синхронизаторы, которые и применяются в данном проекте.

Стоит отметить, что также существуют и другие решения данной проблемы, но все они в основном связаны с передачей большого и непрерывного потока данных. Для данных нужд может быть использован **dual-clock** FIFO буфер. 

## Тактовые домены

### CDC между асинхронными доменами

Под асинхронными доменами понимаются те области, в которых синхросигналы относительно друг друга и сдвинуты по фазе, и отличаются по частоте.

Чтобы обеспечить корректную передачу данных между тактовыми доменами, необходимо добавить управляющее окружение к существующей CDC-логике.

На стороне передатчика реализуется конечный автомат (FSM) с тремя состояниями:

1. LOAD_DATA — состояние загрузки данных в передающий регистр. Данные должны быть установлены раньше сигнала VALID и оставаться стабильными на протяжении всей передачи.
2. LOAD_VALID — состояние установки сигнала валидности данных VALID.
3. WAIT_RX_VALID — состояние ожидания подтверждения со стороны приёмника о завершении обработки данных.

Для организации обратной связи может использоваться синхронизированный сигнал VALID со стороны приёмника. После получения подтверждения передатчик сбрасывает собственный сигнал VALID и переходит в состояние LOAD_DATA.

На стороне приёмника реализуется конечный автомат с двумя состояниями:

1. UPLOAD_DATA — состояние считывания входных данных при наличии сигнала VALID со стороны передатчика.
2. WAIT_TX_INVALID — состояние ожидания сброса сигнала VALID со стороны передатчика. После снятия сигнала автомат возвращается в состояние UPLOAD_DATA.

### CDC между связанными доменами

Под связанными доменами понимаются те области, в которых синхросигналы относительно друг друга исключительно отличаются по частоте (генерируются одним блоком ресинхронизации PLL или MMCM).

Как видно, в данном случае тактовые сигналы отличаются исключительно частотой, сохраняя при этом одинаковую фазу, поскольку формируются от общего PLL/MMCM-блока.

Исходя из представленной схемы, дополнительные сигналы VALID в данной реализации могут не использоваться, поскольку передача данных осуществляется в заранее определённые моменты времени. Благодаря известному взаимному расположению тактовых сигналов вероятность возникновения метастабильных состояний на стороне приёма существенно снижается, а данные могут быть гарантированно захвачены в корректный момент времени. 

Так как на Рисунке выше представлена двусторонняя передача данных, возможно также использование двух экземпляров существующего моста — отдельно для каждого направления обмена.

В рассматриваемой реализации дополнительное управляющее окружение может быть реализовано следующим образом:

* конечный автомат (FSM) со стороны передатчика, учитывающий временные интервалы, через которые допускается повторная передача данных.
* конечный автомат (FSM) со стороны приемника, учитывающий временные интервалы, через которые допускается повторный прием данных с момента последнего приема.

### Дополнительная информация

Очевидно, что подобный способ передачи данных не способен обеспечить производительность, сопоставимую с высокоскоростными интерфейсами передачи данных. Тем не менее, за одну транзакцию может передаваться не только один пакет данных, а сразу несколько посредством расширения разрядности шины данных.

Ширина шины может быть увеличена до разумных пределов, однако при этом необходимо учитывать:

* рост энергопотребления;
* увеличение количества используемых логических ресурсов;
* усложнение трассировки;
* увеличение длины и нагрузки межсоединений;
* и другие связующие факторы.
